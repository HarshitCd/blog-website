# Leetcode 3940: Limit Occurrences in a Sorted Array

## 1. Problem summary
You’re given a sorted array nums and an integer k. Remove extra duplicates so that each distinct number appears at most k times, while preserving the original relative order. Return the resulting array (or its length in the in-place variant).

## 2. What the submitted code does
```go
func limitOccurrences(nums []int, k int) []int {
    m := make(map[int]int)
    for _, num := range nums {
        m[num] = min(m[num]+1, k)
    }

    n := 0
    for _, v := range m {
        n += v
    }

    ans := make([]int, n)
    idx := 0
    for _, num := range nums {
        if m[num] != 0 {
            ans[idx] = num
            idx++
            m[num]--
        }
    }

    return ans
}
```

### 2.1. Explanation

First pass: build a map m that stores, for each value, how many times it should appear in the final array. The code increments a counter capped at k.

Compute `n`, the total number of elements the answer will have, by summing values in the map.

Allocate `ans` of length `n`.

Second pass: iterate nums again; whenever `m[num] > 0`, append the number to `ans` and decrement `m[num]`. Because we iterate nums in order, the relative order is preserved.

### 2.2. Correctness and complexity

Correctness: This produces an array where each distinct value appears at most `k` times and original order is preserved.

Time: `O(n)` expected for the two linear passes and map operations.

Space: `O(u)` where `u` is the number of distinct values (for the map) plus `O(n')` for the output slice ans. This is not in place and may use substantial extra memory when nums is large.
